# android/ — Agent Context

This is the Capacitor-generated Android project. It is a standard Android Studio / Gradle project. The web app is served from local assets — no network connection is needed or used.

## How the Web App Gets In

```
npm run build         → dist/
npx cap sync android  → android/app/src/main/assets/public/
```

`android/app/src/main/assets/public/` is **committed to git** so the Android project builds without requiring Node/npm. Always keep these in sync when pushing changes to the web app.

## Key Files

| Path | Purpose |
|---|---|
| `app/src/main/java/com/sprintread/app/MainActivity.java` | Entry point — extends `BridgeActivity` (Capacitor) |
| `app/src/main/assets/public/` | Built web app (committed) |
| `app/src/main/assets/capacitor.config.json` | Generated from `capacitor.config.ts` by `cap sync` |
| `app/src/main/res/` | Icons, splash screens, layouts |
| `capacitor-cordova-android-plugins/` | Capacitor plugin bridge stubs |

## Package / Bundle ID

`com.sprintread.app` — used in `app/build.gradle` and `AndroidManifest.xml`.

## Gradle Stubs

`capacitor-cordova-android-plugins/cordova.variables.gradle` and `build.gradle` are committed directly. Without them Android Studio shows a "file not found" error. They are normally generated by `npx cap sync` but are committed here to allow building without npm.

## gitignore

`android/.gitignore` excludes build artifacts, keystores, IDE files, and `google-services.json`. The `capacitor-cordova-android-plugins/src/` subdirectory is also excluded (generated Java sources), but the gradle stubs outside `src/` are kept.

## Building for Release

1. `npm run build && npx cap sync android`
2. Open `android/` in Android Studio
3. Build → Generate Signed Bundle/APK
4. Use your own keystore (never commit `.jks` / `.keystore` files)

## Permissions

`AndroidManifest.xml` includes `INTERNET` permission (required by Capacitor WebView even for local assets) and `WAKE_LOCK` (required for the Screen Wake Lock API used during playback).
